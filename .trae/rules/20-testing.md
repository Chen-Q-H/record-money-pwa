# 测试与质量保证规范

## 测试策略

### 测试层级
- **单元测试**: 测试单个函数或组件
- **集成测试**: 测试组件间的交互
- **端到端测试**: 测试完整用户流程

### 测试工具栈
- **测试框架**: Vitest
- **组件测试**: React Testing Library
- **断言库**: Vitest内置断言

## 测试覆盖要求

### 必须测试的功能
- **数据模型**: 所有TypeScript接口和类型定义
- **工具函数**: 所有utils.ts中的函数
- **业务逻辑**: 积分计算、预算检查等核心逻辑
- **组件渲染**: 主要组件的渲染和交互

### 测试用例设计
```typescript
// 示例：积分计算测试
describe('积分计算', () => {
  it('应该为每笔消费计算1积分', () => {
    const expense = { amount: 100, category: '餐饮' };
    expect(calculatePoints(expense)).toBe(1);
  });
  
  it('应该限制每日积分上限为10', () => {
    const expenses = Array(15).fill({ amount: 100, category: '餐饮' });
    expect(calculateDailyPoints(expenses)).toBe(10);
  });
});
```

## 质量保证流程

### 代码质量检查
- **TypeScript检查**: `npx tsc --noEmit`
- **ESLint检查**: `npm run lint`
- **代码格式化**: Prettier自动格式化

### 检查时机
- **提交前**: 每次提交前必须通过所有检查
- **合并前**: PR合并前必须通过CI检查
- **发布前**: 发布前进行完整测试
