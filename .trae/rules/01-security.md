# 安全与数据规范

## 数据安全原则

### 数据存储安全
- **本地存储**: 所有用户数据存储在浏览器localStorage中
- **无敏感信息**: 不存储密码、支付信息等敏感数据
- **数据加密**: 重要数据使用简单加密（如base64编码）

### 数据验证
- **输入验证**: 所有用户输入进行类型和范围验证
- **金额处理**: 金额使用number类型，避免浮点精度问题
- **日期处理**: 使用ISO标准日期格式

## 数据完整性

### 数据模型约束
```typescript
// 消费记录必须满足以下约束
interface ExpenseRecord {
  id: string;           // 唯一标识
  amount: number;       // 金额 > 0
  category: string;     // 有效分类
  date: string;         // ISO日期格式
  description?: string; // 可选描述
}

// 预算设置约束
interface Budget {
  category: string;     // 有效分类
  amount: number;       // 金额 >= 0
}
```

### 业务规则
- **积分计算**: 每次记账获得1积分（与金额无关）
- **等级计算**: 基于记账次数分段（新手0-9次，达人10-19次，大师20+次）
- **预算检查**: 消费金额不能为负数

## 错误处理规范

### 异常处理
- **用户友好错误**: 所有错误信息对用户友好
- **错误边界**: 组件使用错误边界处理渲染错误
- **数据恢复**: 数据损坏时提供默认值恢复

### 边界情况处理
- **空数据**: 正确处理空数组和空对象
- **无效输入**: 拒绝无效的用户输入
- **存储失败**: 处理localStorage写入失败
