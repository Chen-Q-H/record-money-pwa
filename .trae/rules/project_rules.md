# 项目开发规则 - Spec Coding 流程

## Spec Coding 核心理念

### 万物皆 Markdown
- **规范即代码**：所有需求、设计、验收标准都以 Markdown 形式记录
- **AI 驱动开发**：AI 读取 Markdown 规范，生成/修改代码实现
- **持续规范修复**：开发过程就是不断修复和完善规范的过程

### 开发流程
1. **需求分析** → 在 `specs/` 目录创建功能规格文档
2. **设计规范** → 更新 `CLAUDE.md` 和相关规则文档
3. **代码实现** → AI 读取规范生成代码
4. **验证测试** → 执行代码质量检查
5. **规范修复** → 根据实现结果更新规范

## 技术环境配置

### Node.js 版本要求
**重要**: 每次开启新的控制台时，必须先切换到 Node.js 22 版本

#### 切换命令
```bash
fnm use 22
```

#### 验证命令
```bash
node --version
```

#### 预期版本
- 当前项目需要 Node.js v22.21.1 或更高版本
- 默认的 Node.js v14.21.3 会导致语法错误

#### 错误示例
如果使用旧版本，会出现以下错误：
```
SyntaxError: Unexpected token '??='
```

### 开发服务器启动
启动开发服务器时，建议使用以下命令：
```bash
npm run dev -- --force
```

## 代码质量检查流程

### 检查时机（规范修复环节）
- 创建新组件或修改现有组件后
- 修改 TypeScript 类型定义后
- 修改配置文件后
- 提交代码前

### 检查命令
```bash
# TypeScript 类型检查
npx tsc --noEmit

# ESLint 代码规范检查
npm run lint
```

### 检查标准
- TypeScript 编译无错误
- ESLint 检查通过（0错误，0警告）
- 代码符合 React 19 纯函数规则
- 符合项目设计规范

## 项目架构规范

### 技术栈
- **前端框架**: React 19.2.5 + TypeScript 6.0.2
- **构建工具**: Vite 8.0.10
- **样式方案**: CSS Modules + 小猪主题配色
- **PWA 支持**: Vite PWA 插件

### 目录结构规范
```
src/
├── components/     # 可复用组件
├── pages/         # 页面组件
├── types.ts       # TypeScript 类型定义
├── utils.ts       # 工具函数
└── App.tsx        # 应用入口

specs/             # 功能规格文档（Markdown）
.trae/rules/       # 开发规则和配置
CLAUDE.md          # 项目主规范
```

## 功能模块规范

### 当前功能模块
1. **首页** (`HomePage`): 快速记账、支出统计、预算概览
2. **预算管理** (`BudgetPage`): 分类预算设置和超支提醒
3. **花销记录** (`ExpensesPage`): 按日期分组的消费明细查看
4. **商店** (`StorePage`): 积分兑换背景和主题皮肤

### 设计主题规范
- **配色方案**: 小猪粉色主题（#FFB6C1 等）
- **移动优先**: 适配移动端 PWA 全屏体验
- **用户友好**: 简洁直观的交互设计
